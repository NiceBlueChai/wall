# Wall v1 UX / IxD 设计规范

## 文档职责

本文档是 Wall v1 的行为契约，说明用户如何完成任务、界面如何响应，以及失败后如何恢复。视觉尺寸、颜色、
组件和图标继续以 `docs/design/wall-v1-design-spec.md` 与 Figma 文件为准。

- Figma 文件：`SdCRz7VSCFwVC3Apk9dniq`。
- UX / IxD 页面：`06 UX & Flows`（`166:2`）；完整内容容器为 `166:3`。
- 六个交付区块：Scope `166:4`、Journey `166:5`、State `166:6`、Interaction `166:7`、
  Recovery `166:8`、Screen Index `166:9`。
- 产品范围：Windows 10/11 x64、完全离线、本地视频与图片。
- 设计边界：只覆盖当前 v1 能力，不包含账号、云同步、在线商店、播放列表或场景编辑器。
- 数据原则：移出壁纸库、删除分类和清理失效项均不删除或修改源文件。

## 体验目标与验收指标

### 核心目标

1. 首次使用者无需阅读说明即可在“导入 → 选择目标 → 设为壁纸”三段任务中完成首次播放。
2. 用户始终能分辨媒体预览、桌面播放和托盘状态，图片不出现视频专属控制。
3. 多显示器配置先编辑草稿，只有点击“应用”才提交；取消不改变当前播放布局。
4. 所有移除操作都在确认前说明影响范围，并明确源文件会保留。
5. 异步操作立即提供反馈；错误保留当前上下文，并给出可执行的恢复动作。

### 完成标准

- 任何主任务最多经过一个主导航入口；常用播放动作不藏在二级菜单中。
- 用户触发操作后立即出现按下、忙碌或状态变化反馈，不允许无反馈等待。
- 所有弹窗可用键盘完成，并在关闭后把焦点返回触发控件。
- 运行、暂停、空闲、部分暂停、错误和离线状态同时使用文字与形状，不只依赖颜色。
- Figma 中每条核心旅程均有起点、触发、系统响应、成功结果、失败恢复与取消路径。

## 信息架构

Wall 保留两个一级入口：

- 壁纸库：浏览、搜索、分类、导入、详情、显示目标、播放和库管理。
- 设置：常规、播放、性能、关于。

系统托盘是后台运行时的快捷入口，不是第三套信息架构。托盘与主窗口读取同一播放状态，并调用同一组操作。

### 壁纸库页面职责

- 顶部第一组：搜索与媒体类型筛选。
- 顶部第二组：显示目标、库管理、导入壁纸。
- 左侧分类只负责筛选与分类维护，不重复放置批量管理入口。
- 卡片单击打开详情；卡片悬停或键盘聚焦时显示“设为壁纸”快捷操作。
- 不使用双击触发播放。双击会先触发单击导航，结果不可预测，也不利于触屏板和辅助技术。
- 批量模式下，卡片单击、Enter 或 Space 只切换选择，不打开详情或播放。

### 壁纸详情页面职责

- 视频：提供应用内视频预览和原生预览控制；预览默认静音、循环播放，不影响桌面壁纸状态。
- 图片：只显示静态预览，不显示继续、暂停、进度、音量或静音控件。
- “设为壁纸”应用到当前已提交的显示目标。
- “暂停 / 继续”和“停止”只控制正在使用当前壁纸的目标，不误伤其他显示器上的壁纸。
- “从库中移除”使用低强调危险入口；真正危险强调只出现在确认弹窗的提交按钮。

### 设置页面职责

- 常规：开机启动、关闭到托盘、恢复上次壁纸、界面语言。
- 播放：全局画面与视频声音默认值。
- 性能：自动暂停条件。
- 关于：版本、平台、许可证和项目主页。
- 设置项即时保存。保存失败时控件回到服务端快照值，并显示持续错误，不伪造成功状态。

## 核心用户旅程

### 旅程 A：首次导入并播放

| 阶段 | 用户动作 | 界面响应 | 取消或失败 |
| --- | --- | --- | --- |
| 空库 | 点击“导入壁纸” | 打开 Windows 文件选择器，保留主窗口上下文 | 取消选择后原地返回，不显示错误 |
| 选择文件 | 选择一个或多个支持文件 | 关闭选择器后显示不可交互的导入中弹窗和不确定进度 | 读取失败时关闭忙碌态，保留已存在媒体并显示错误 |
| 导入完成 | 无需额外确认 | 新项目出现在库中，显示成功 Toast | 不支持格式时列出原因，源文件不变，可重新选择 |
| 选择目标 | 打开显示器选择器 | 使用当前布局创建草稿，展示在线、主屏与离线状态 | 取消、Esc 或外部点击丢弃草稿 |
| 提交目标 | 点击“应用” | 按钮进入忙碌态；成功后关闭弹层并更新目标摘要 | 失败时弹层保持打开，草稿保留并显示错误 |
| 播放 | 打开详情后点击“设为壁纸”，或使用卡片快捷操作 | 目标立即进入忙碌反馈，成功后卡片、详情、侧栏和托盘同步 | 失败时保留目标配置，显示“重试”和可理解错误 |

### 旅程 B：浏览、预览与调整

1. 用户可组合分类、媒体类型和搜索词；三者同时生效。
2. 空库、分类为空和筛选无结果使用不同文案与恢复动作。
3. 打开视频详情后，可在应用内播放预览；预览播放不改变桌面壁纸，也不继承全局音量。
4. 壁纸级设置即时保存，并以“使用全局设置 / 已单独设置”明确继承关系。
5. “恢复全局设置”一次清除该壁纸所有覆盖值，不改变全局设置。
6. 返回壁纸库时保留搜索、筛选和分类上下文；从已移除项目返回时回到全部壁纸。

### 旅程 C：配置多显示器

| 模式 | 最小选择 | 提交结果 | 不可用情况 |
| --- | ---: | --- | --- |
| 独立 | 1 块在线屏幕 | 当前壁纸只作用于所选屏幕 | 离线屏幕不可作为新目标 |
| 复制 | 2 块在线屏幕 | 组内播放同一壁纸并共享一个音源 | 在线屏幕少于 2 块时禁用并说明原因 |
| 铺展 | 2 块在线屏幕 | 一个播放器覆盖所选屏幕的联合区域 | 在线屏幕少于 2 块时禁用并说明原因 |

- 打开选择器时复制已提交布局形成草稿。
- 切换到独立模式时只保留一个目标；切换到复制或铺展时不自动提交。
- “应用”仅在选择数量合法时可用。
- 屏幕断连后保留既有分配并标记离线；重新连接时按原分配恢复。
- 离线期间允许查看旧分配，但不能把离线屏幕加入新的布局。

### 旅程 D：管理壁纸库

#### 分类管理

- 创建和重命名要求 1 至 40 个非空白字符；错误显示在输入框附近，弹窗不关闭。
- 删除分类需要确认；壁纸库条目和源文件均保留。
- 删除当前分类后回到“全部壁纸”，不保留失效分类筛选。

#### 批量管理

- 从“管理 → 批量管理”进入，初始选择为空。
- 未选择项目时，分类操作和“移出库”均禁用。
- 添加到分类时，全部已包含的目标分类禁用并标注“已包含”。
- 从分类移除时，全部未包含的目标分类禁用并标注“未包含”。
- 关闭分类菜单不清空选择；取消批量管理才清空选择并退出模式。
- 批量移出需二次确认。若包含运行中壁纸，确认框显示受影响项数，并说明相关屏幕会停止。

#### 清理失效项

1. 点击“管理 → 清理失效项”后重新扫描整个壁纸库，不能复用旧 `missing` 状态。
2. 扫描期间禁用管理入口并显示忙碌反馈，不阻塞已运行的壁纸。
3. 零失效项只显示 Toast，不打开空确认框。
4. 发现失效项后显示数量、运行中数量和“源文件不会被删除或修改”。
5. 确认执行时再次扫描；扫描结果变化时以最终移除数量为准。
6. 取消后不改变库、选择或播放状态。

### 旅程 E：后台与托盘控制

- `关闭时最小化到托盘 = 开`：点击关闭隐藏主窗口，播放继续；不弹出重复确认。
- `关闭时最小化到托盘 = 关`：点击关闭退出应用，播放器随应用停止。
- 单击托盘图标打开并聚焦主窗口；右键打开原生托盘菜单。
- 托盘按空闲、运行、暂停显示同尺寸轮廓和不同状态角标。
- 多目标混合状态使用运行图标，并在菜单信息项显示“部分暂停”。
- 无运行目标时，暂停、静音、停止和目标子菜单禁用。
- 托盘操作完成后，主窗口下次显示时必须反映同一快照，不额外维护 UI 状态。

## 状态模型

### 桌面播放状态

```text
空闲 --设为壁纸成功--> 运行
运行 --手动暂停或任一自动暂停原因出现--> 暂停
暂停 --移除手动暂停且没有其他暂停原因--> 运行
运行/暂停 --停止或移除当前壁纸--> 空闲
运行/暂停 --播放器失败--> 错误
错误 --重试成功--> 运行
错误 --停止--> 空闲
```

- 暂停原因是集合，不是单个布尔值。手动继续不能覆盖全屏、电池、最大化或显示器休眠导致的自动暂停。
- 多显示器状态按目标计算；全局摘要可为“全部运行”“部分暂停”“全部暂停”或“错误”。
- 图片使用“正在显示 / 已停止”，不出现视频的暂停、音量和静音语义。

### 操作生命周期

所有异步写操作统一使用以下状态：

```text
就绪 -> 提交中 -> 成功
              -> 可恢复错误 -> 重新提交
              -> 可恢复错误 -> 取消并返回原状态
```

- 提交中禁用重复提交及会破坏上下文的关闭操作。
- 成功 Toast 默认显示 3 秒，不抢夺焦点。
- 错误不能只用瞬时 Toast；在相关弹窗或页面内保留错误，直到重试、取消或下一次成功操作。
- 批量操作使用原子语义：失败时不显示部分成功，也不清空选择。

## 交互规则

### 层级与关闭

| 层级 | 打开方式 | Esc | 点击外部 | 完成后焦点 |
| --- | --- | --- | --- | --- |
| 下拉菜单 | 点击触发按钮 | 关闭 | 关闭 | 返回触发按钮 |
| 显示器弹层 | 点击目标按钮 | 取消草稿并关闭 | 取消草稿并关闭 | 返回目标按钮 |
| 普通弹窗 | 点击明确操作 | 关闭 | 不关闭 | 返回触发按钮 |
| 危险确认 | 点击危险入口 | 等同“取消” | 不关闭 | 取消后返回入口；成功后移到合理页面标题 |
| 导入中 | 文件选择完成 | 不关闭 | 不关闭 | 完成后返回导入按钮或首个新卡片 |

- 同一时刻只允许一个菜单、弹层或弹窗处于顶层。
- 打开一个顶部弹层时关闭其他顶部弹层，避免管理菜单与显示器选择器重叠。
- 弹窗打开后锁定背景滚动并设置焦点陷阱。

### 键盘与焦点

- Tab / Shift+Tab 按视觉顺序移动焦点。
- Enter 或 Space 激活按钮、卡片快捷操作、开关和复选项。
- 菜单使用上/下方向键移动，Home / End 到首尾，Enter 选择，Esc 关闭。
- 分段控件使用左/右方向键切换，选中项具有 `aria-pressed` 或等价语义。
- 危险确认默认聚焦“取消”；创建和重命名分类默认聚焦文本输入框。
- 所有焦点环至少 2 px，并同时使用轮廓与明暗差异。

### 文案与危险等级

- “删除”只用于真实删除分类；媒体记录使用“从库中移除”或窄布局中的“移出库”。
- 危险入口采用低强调样式，确认按钮才使用实心危险色，避免工具栏被红色按钮主导。
- 所有确认文案必须包含对象、数量、对运行目标的影响及源文件处理方式。
- 取消永远不更改选择、草稿、播放或媒体库状态。

## 媒体类型差异

| 能力 | 视频 | 图片 |
| --- | --- | --- |
| 应用内预览 | 播放、暂停、进度、预览音量；默认静音 | 静态查看 |
| 桌面状态 | 运行、暂停、静音、停止 | 正在显示、停止 |
| 帧率 | 显示 | 隐藏 |
| 硬件解码 | 显示 | 隐藏 |
| 静音与音量 | 显示 | 隐藏 |
| 缩放、画幅、抗锯齿 | 显示 | 显示 |

预览控制与桌面控制必须分区并使用不同文案，避免用户误以为暂停预览会暂停桌面壁纸。

## 空态、错误与恢复

| 状态 | 主文案 | 主恢复动作 | 保留内容 |
| --- | --- | --- | --- |
| 空壁纸库 | 还没有壁纸 | 导入壁纸 | 设置与显示布局 |
| 分类为空 | 此分类还没有壁纸 | 进入批量管理 | 当前分类 |
| 搜索无结果 | 没有搜索结果 | 清除搜索与筛选 | 当前分类 |
| 文件丢失 | 文件丢失 | 定位新文件 | 分类、覆盖设置、显示分配 |
| 播放失败 | 无法播放此壁纸 | 重试 | 当前目标与壁纸设置 |
| 显示器离线 | 显示器已断开 | 等待重连或选择在线屏幕 | 原显示分配 |
| 项目已移除 | 找不到这张壁纸 | 返回壁纸库 | 全局筛选重置为全部 |

- “定位新文件”只允许选择一个文件；成功后重新读取媒体信息并清除失效状态。
- 选择错误类型或无法读取的文件时保留原路径，允许再次选择或从库中移除。
- 错误信息使用用户可理解的结果和恢复动作；日志属于次要诊断入口，不替代错误说明。

### `06 UX & Flows`

页面只承载行为，不复制完整 UI。使用现有变量、文本样式、组件和图标，组织为以下六个区块：

1. Scope & Principles：范围、目标、非目标和全局原则。
2. Journey Map：首次播放、浏览预览、多显示器、库管理、托盘五条主旅程。
3. State Models：播放、显示布局草稿、异步操作与媒体丢失状态。
4. Interaction Contract：弹层、弹窗、键盘、焦点、忙碌、成功与错误规则。
5. Recovery Matrix：导入、播放、离线、移除和设置保存失败的恢复路径。
6. Screen Index：30 张正式画板的分组索引与关键跳转关系。

### 原型连接

- 原型只连接能验证主任务的关键路径，不为每个重复按钮创建连线。
- 起点为“壁纸库 / 标准状态” `93:422`，流程名为“Wall v1 主流程”。
- 至少可走通：导入、打开详情并预览、选择显示布局、批量移除、失效清理、单项移除和设置切换。
- 确认弹窗的取消返回来源状态；提交连接到成功后的稳定状态，错误通过同页注释说明。

# Wall v1 设计规范

## 产品范围

Wall 是面向 Windows 10/11 x64 的免费开源动态壁纸工具。v1 完全离线，不包含账号、商城、订阅、评分、
上传或在线推荐。用户导入本地视频或图片后，可以设置、暂停、静音或停止壁纸。

## 设计原则

- 本地优先：所有媒体、缩略图和设置保存在本机。
- 桌面优先：界面适配鼠标、键盘、系统托盘和 Windows 窗口行为。
- 低干扰：使用深色单主题，减少动态壁纸应用在后台运行时的视觉干扰。
- 状态明确：始终显示当前壁纸以及运行、暂停、静音或错误状态。
- 最少入口：主导航只保留“壁纸库”和“设置”，不展示不可用的在线入口。

## 窗口与布局

- 标准设计尺寸：1280 × 800 px。
- 最小支持尺寸：960 × 640 px。
- 左侧导航宽度：208 px；顶部标题栏高度：48 px。
- 内容区使用 24 px 外边距和 16 px 网格间距。
- 主窗口关闭按钮默认隐藏到托盘，而不是退出应用；可在设置中改变。

## 信息架构

### 壁纸库

- 标题、搜索框、媒体类型筛选和“导入壁纸”主按钮。
- 用户分类侧栏显示分类数量，并支持创建、重命名和删除；删除分类不会删除媒体或源文件。
- 批量模式支持多选壁纸，并将所选项目原子添加到或移出指定分类。
- 右上角显示器入口用于选择独立、复制或铺展目标；离线显示器保留分配但不能应用新布局。
- 壁纸卡片网格；卡片显示预览、名称、视频/图片类型和运行状态。
- 卡片单击打开详情，双击立即设为壁纸。
- 空库状态直接提供导入入口，不展示在线内容。

### 壁纸详情

- 大尺寸媒体预览。
- 文件名、类型、分辨率、时长和本地路径。
- 主操作：设为壁纸、暂停/继续、停止。
- 次操作：静音/取消静音、打开文件位置、从库中移除。
- 视频可以覆盖全局缩放、画幅、抗锯齿、帧率、硬件解码、静音和音量；图片隐藏视频专属选项。
- 覆盖设置为空时使用全局设置；“恢复全局设置”一次清除当前壁纸的全部覆盖值。

### 设置

- 常规：开机启动、关闭时最小化到托盘、启动后恢复上次壁纸、界面语言。
- 播放：缩放方式、帧率上限、硬件解码、默认静音、音量。
- 性能：全屏应用运行时暂停、使用电池时暂停、显示器休眠时暂停。
- 关于：版本、开源许可证、项目主页。

四个设置页分别使用独立的 1280 × 800 完整界面，共用标题栏、侧栏和“常规 / 播放 / 性能 / 关于”页签。
切换页签时只替换内容区，窗口外壳、当前壁纸状态和导航位置保持不变。
内容区沿用其他界面的颜色层级：画布使用 `color/surface/canvas`，`color/surface/panel` 仅覆盖实际内容并贴合内容高度，
不使用固定高度制造大面积空白色块。版本、平台和离线状态只在“关于”页展示。

#### 常规设置

- “启动与窗口”面板提供开机启动、关闭时最小化到托盘、恢复上次壁纸三个开关和界面语言选择。
- 常规页不显示版本信息或离线状态，避免与“关于”页重复。

#### 播放设置

- “播放”面板提供 Cover、Contain、Stretch，原始/屏幕及常用画幅，关闭/均衡/高质量抗锯齿，
  源帧率/24/30/60 FPS、硬件解码、默认静音和壁纸音量。
- 自动暂停相关设置不放在本页，统一归入性能设置，避免同一配置在两个页面重复出现。

#### 性能设置

- “自动暂停”面板提供全屏应用运行时、最大化应用运行时、使用电池时、显示器休眠时四个暂停开关。
- 每个开关都显示触发条件和行为说明；v1 不增加进程白名单、单应用规则或 GPU 阈值等高级配置。

#### 关于设置

- 显示应用图标、Wall 版本、Windows 10/11 x64、免费开源和完全离线说明。
- 显示作者 `NiceBlueChai` 和联系邮箱 `bluechai@qq.com`；标签左对齐，值保持同一列。
- 提供“查看开源许可证”和“打开项目主页”两个操作；许可证读取本地文件，项目主页仅在用户主动点击时交给
  系统默认浏览器处理，应用自身不请求网络。
- 项目主页固定为 `https://github.com/NiceBlueChai/wall`，按钮保持可用，不在应用内嵌网页。
- 系统无法打开本地许可证或浏览器时显示错误提示，不影响壁纸播放。

## 缩放与多显示器行为

- Cover（默认）：保持宽高比并放大到覆盖整个屏幕，超出部分居中裁切。
- Contain：保持宽高比并完整显示，空余区域使用黑色填充。
- Stretch：拉伸到屏幕尺寸，不保持宽高比。
- 独立：每台选中显示器拥有独立播放实例和壁纸分配。
- 复制：一组显示器联动播放同一壁纸，每组只有一个音源，播放漂移超过 100 ms 时校准。
- 铺展：一个播放器覆盖选中显示器在 Windows 虚拟桌面中的联合像素区域。
- 显示器临时离线时保留分配，重新连接后恢复。

## 状态与错误

- 空壁纸库：说明支持本地视频和图片，并显示导入按钮。
- 导入中：对话框显示文件名和进度，不阻塞已有壁纸播放。
- 不支持格式：保留原文件不变，提示支持的格式并允许重新选择。
- 文件丢失：卡片显示错误状态，可定位新文件或从库中移除。
- 播放失败：显示简短错误、重试和打开日志目录操作。

## 系统托盘菜单

菜单从上到下固定为：

1. 打开 Wall
2. 当前壁纸名称（禁用的信息项；无壁纸时显示“未运行壁纸”）
3. 全部屏幕暂停/继续
4. 全部屏幕静音/取消静音
5. 全部屏幕停止
6. 按显示器或联动组操作的子菜单
7. 分隔线
8. 开机启动（复选项）
9. 分隔线
10. 退出 Wall

运行、暂停和无壁纸状态分别使用不同托盘图标。暂停、静音和停止项在无壁纸时禁用。单击托盘图标打开主窗口；
右键打开菜单。

- Slider：默认、悬停、禁用。
- SegmentedControl：Cover、Contain、Stretch，以及独立、复制、铺展。
- WallpaperCard：默认、悬停、选中、批量选中、运行、错误。

cover、contain、stretch、folder、trash、more、close、minimize、maximize、restore、back、check、chevron-down、
chevron-right、warning、info、video、image、power、external-link。

## 视觉令牌

- 单一深色模式，基础背景从 `#080A12` 到 `#202642`。
- 品牌紫色为 `#7C5CFF`，悬停状态为 `#9278FF`。
- 成功、警告、危险分别为 `#38D996`、`#FFB84D`、`#FF5D73`。
- 字体使用 Noto Sans SC，确保中文在普通画板和嵌套组件中稳定渲染；正文最小字号 12 px，常用正文 14 px，
  页面标题 24 px。
- 间距使用 4 px 基线；圆角使用 4、8、12、16 px。

## 可访问性

- 正文与背景对比度至少满足 WCAG AA。
- 所有交互控件具有悬停、聚焦和禁用状态；键盘焦点环不得只依赖颜色变化。
- 主要点击区域不小于 32 × 32 px。
- 图标按钮必须在实现中提供可访问名称和工具提示。
- 状态同时使用文字或形状表达，不能只依赖颜色。

## Tauri 实现映射

- Figma 变量名直接映射为 CSS 自定义属性，例如 `color/accent/default` 对应
  `var(--color-accent-default)`。
- Figma 组件名与前端组件名保持一致；Rust 只负责窗口、托盘、媒体进程和系统集成。
- 缩放方式使用枚举 `cover | contain | stretch`，默认值为 `cover`。
- 显示模式使用枚举 `independent | clone | span`；复制至少选择两个屏幕，铺展按 Windows 虚拟桌面
  像素联合区域计算。
- 壁纸设置使用可空覆盖字段，运行时按“全局设置 → 壁纸覆盖”合并，不复制一份全局默认值。
- 托盘菜单命令与主界面调用同一组 Tauri commands，避免两套状态逻辑。
- Code Connect 在首批前端组件落地后添加；空代码库阶段不创建无效映射。

## 非目标

- 在线壁纸商城、账号、云同步、评分、订阅和内容审核。
- 场景编辑器、网页壁纸和互动壁纸。
- 播放列表和定时切换。
- macOS、Linux 和 Windows ARM64。

# 分类编辑交互设计

## 目标

让详情页和批量管理在“已有分类”和“尚未创建分类”两种状态下都能完成分类操作，并将详情信息卡重新对齐
Figma 的层级、间距和分类组件。分类操作只改变库内归属，不修改或删除源文件。

## 问题与设计原则

- 症状：详情页点击“编辑分类”后，在没有分类时出现空菜单；批量分类菜单存在相同问题。
- 原因：菜单只循环现有分类，没有空状态、恢复动作或创建入口；当前详情信息顺序和控件尺寸也偏离 Figma。
- 原则：管理入口在任何数据状态下都必须提供可完成的下一步；空状态不能只显示空容器。
- 修复范围：详情页分类区、批量添加/移除分类菜单，以及两处复用的分类创建弹窗，按同一类交互修复。

## 详情页交互

详情卡沿用已确认的 384 × 324 容器、上方信息区整体滚动和底部操作固定。上方内容按以下顺序排列：

1. 壁纸名称。
2. 媒体类型、格式、尺寸和时长。
3. 完整文件路径；只隐藏 Windows 内部的 `\\?\` 前缀，不截断路径。
4. “分类”标题和分类标签区。
5. 当前显示目标与播放状态。

分类区行为：

- 已归属分类使用现有 28px `Category Tag`，标签末尾提供移除图标；移除不二次确认。
- 标签区末尾使用轻量的“＋ 添加分类”操作，不再使用当前独立的“编辑分类”按钮。
- 点击“添加分类”打开 208px 分类选择菜单；已有归属项显示勾选并禁用，其余项可直接添加。
- 菜单底部固定显示“新建分类”，与已有分类列表用分隔线区分。
- 全局尚无分类时，菜单显示“还没有分类”及主动作“新建并添加”，不得渲染空菜单。
- 新建成功后自动将新分类加入当前壁纸并关闭弹窗与菜单。
- 创建成功但自动加入失败时保留已创建分类，显示“分类已创建，但添加到当前壁纸失败，请重试”，重新打开菜单即可添加。
- 分类请求期间禁用分类标签、添加入口和菜单项；成功后焦点返回“添加分类”，失败后焦点回到原操作。

## 批量管理交互

- “添加到分类”复用同一选择菜单和空状态；新建成功后自动加入当前选中的全部壁纸。
- “从分类移除”在全局无分类时禁用；有分类但所选壁纸均未归属时，菜单项显示“未包含”并禁用。
- 创建或分类变更期间保留当前批量选择；关闭菜单也不清空选择。
- 批量创建成功但自动加入失败时采用与详情页相同的部分成功说明，分类本身不回滚。

## 分类创建弹窗

- 详情、批量和侧栏复用同一弹窗外观、1–40 个 Unicode 字符校验、忙碌状态与错误文案。
- 从详情或批量菜单进入时标题为“新建分类”，确认按钮为“创建并添加”。
- 从侧栏进入时保持“创建分类”和“确认”，只创建分类，不自动修改任何壁纸归属。
- 打开后聚焦输入框；取消或 Escape 不改变分类和壁纸归属；保存期间禁止关闭与重复提交。

## 视觉与 Figma

- 修改现有视频详情 `105:1395` 和图片详情 `105:1531`，不新增重复的顶层页面。
- 详情卡统一使用 20px 内边距；标题、媒体信息、路径和分类标题恢复 Figma 的纵向顺序。
- 分类标签复用 `Category Tag`；分类选择菜单复用 `Menu Item`，行高 36px，并增加默认、空状态和创建入口变体。
- 保留已确认的完整路径、信息区单一滚动和底部固定播放操作，这三项优先于旧详情稿中的静态布局。
- 批量分类菜单同步更新现有组件与相关状态稿，不创建新的图标、颜色变量或重复组件。

## 错误、可访问性与验收

- 分类菜单支持方向键、Home、End、Escape、外部点击关闭和触发器焦点返回。
- 创建弹窗保持焦点陷阱；错误显示在输入框附近，并使用现有内联错误样式。
- 自动化覆盖详情有分类、详情无分类、批量无分类、创建并自动加入、部分成功和失败重试。
- 最终截图覆盖视频详情有分类、图片详情无分类、分类菜单空状态和批量菜单空状态。
- 验收标准：用户不离开当前任务即可创建并归类；任何分类入口都不会打开空白菜单；详情卡与 Figma 对齐且无重叠。

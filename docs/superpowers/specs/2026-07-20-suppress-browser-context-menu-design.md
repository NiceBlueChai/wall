# 禁止非编辑界面的浏览器右键菜单

## 目标

应用普通界面右键时不显示 Chromium 菜单；输入控件仍保留剪切、复制和粘贴等编辑菜单。

## 实现

在 `App.vue` 的 `.app-window` 根容器统一处理 `contextmenu`。事件来自 `input`、`textarea` 或
`contenteditable` 元素时不拦截，其他目标调用 `preventDefault()`。

## 边界

不创建 Wall 自定义右键菜单，不修改 Figma、视频预览或现有业务操作。

## 验证

组件测试分别派发可取消的右键事件，证明普通界面事件被阻止、输入框事件未被阻止；随后运行完整前端测试与构建。

# 视频壁纸窗口层级稳定性设计

## 问题与契约

在 Windows 11 的 raised desktop 路径中，复制模式会为每块显示器创建一个 mpv 窗口。当前每个窗口每
250 ms 独立执行一次 `SetWindowPos`，但多个窗口都要求紧跟在 `SHELLDLL_DefView` 后，形成无法同时满足的
层级竞争，短暂露出 Explorer 绘制的原 Windows 背景。

用户可以在独立、复制和铺展模式持续播放视频壁纸；任何同时运行的播放器窗口都不能互相争抢桌面层级，
也不能在层级维护时露出系统背景。Explorer 刷新后仍应自动恢复正确层级。

## 设计

采用一个进程级层级维护者，由 `MpvPlayerManager` 统一拥有。管理器收集全部 raised desktop 播放器窗口，
按稳定顺序组成 `SHELLDLL_DefView -> 播放器 1 -> 播放器 2` 的链，并由单线程周期性维护。播放器新增、停止或
替换时重建这条链；`MpvPlayer` 不再各自启动维护线程。

不把复制模式改成跨屏单窗口，因为不同分辨率显示器需要独立缩放；不取消周期维护，因为 Explorer 可能重建
桌面层级；不增加全系统 mpv 扫描和终止逻辑，避免误伤用户自行运行的 mpv。

现有孤儿 mpv 进程仅在本次验证前按已确认的 PID 和可执行路径一次性清理。正常退出仍沿用
`MpvPlayerManager::drop`；异常强杀的进程生命周期治理不扩展到本次闪动修复。

## 测试

先添加一个失败测试，要求同一桌面宿主下的多个播放器形成稳定链，而不是全部直接插到图标层之后。实现后运行
播放器 Rust 测试、完整 Cargo 测试、前端测试和构建，最后在两块异分辨率显示器的复制模式中确认视频连续且
不再露出 Windows 背景。

# Wall v1 实现报告

## 当前实施契约（Figma 冻结版）

声明范围：本轮实现覆盖分类与壁纸库管理、壁纸级设置、多显示器独立/复制/铺展、完整播放与性能设置、
同源托盘操作、持久化迁移和对应自动化证明。冻结设计总览为 Figma `115:1747`。

| 层/模块             | 所有者类别     | 当前假设                  | 新契约                           | 代码动作 | 证明                             |
| ------------------- | -------------- | ------------------------- | -------------------------------- | -------- | -------------------------------- |
| Vue 类型/store/API  | 必需数据/状态  | 单一媒体筛选与全局设置    | 分类、批量选择、继承、多屏目标   | 实现     | Vitest store/API                 |
| Vue 页面/组件/CSS   | 必需呈现       | 主屏库、简化详情和设置    | 严格映射 30 张正式状态           | 实现     | 组件测试、逐页截图               |
| Rust model/core     | 必需执行与状态 | 单播放器、主屏、全局设置  | 分类、多屏组、覆盖设置、暂停叠加 | 实现     | Cargo 单元/集成测试              |
| Rust storage        | 必需数据/状态  | `library.json` 为媒体数组 | 兼容旧数据的库与分类持久化       | 实现     | 旧格式迁移测试                   |
| Rust player/monitor | 必需原生执行   | 一个主屏 mpv              | 独立/复制/铺展与按屏暂停         | 实现     | 参数、几何、状态机测试；实机矩阵 |
| Rust tray/commands  | 必需执行与呈现 | 全局单播放器菜单          | 全屏与按显示器/联动组操作        | 实现     | 命令与菜单状态测试               |
| 文档/发布           | 必需证明       | 旧主屏范围                | 新契约、节点映射、便携包         | 实现     | 构建和 ZIP 内容检查              |

用户可以创建、重命名和删除分类，批量增删壁纸分类，从库中移除单项或多项，清理失效记录，
选择显示器或联动组并应用壁纸及覆盖设置。
用户不能通过删除分类删除媒体源文件，也不能在少于两台在线显示器时启用复制或铺展。

本轮保持单工作区串行实现：项目规则禁止未获用户要求时启动子代理，且 Vue/Rust/契约文件共享序列化模型，
串行 TDD 能避免多个工作树同时修改同一协议。Windows 原生行为以真实 mpv/WorkerW 为最终边界；普通 Vitest 和
Cargo 测试中的窗口、托盘、显示器枚举属于可重复的替代证明，发布前仍执行真实 Windows 10/11 矩阵。

## 已实现契约

- Windows 10/11 x64、完全离线、本地视频与图片、MIT 源码和便携 ZIP。
- Vue 3 四页设置、分类壁纸库、批量管理、单项/批量移除、失效清理、视频/图片详情、搜索筛选和统一提示。
- 库卡片单击进入详情、独立快捷按钮设置壁纸；批量模式只改变选择，不触发播放或导航。
- 显示布局先编辑草稿再提交；取消、Esc 和外部点击不改变现有目标，保存失败保留草稿供重试。
- 视频支持软件内静音预览；图片不显示暂停、音量、帧率、硬件解码或预览播放入口。
- 写操作统一具备忙碌防重复、失败就地恢复；菜单和弹窗支持完整键盘与焦点闭环。
- 分类、显示布局、全局设置和壁纸覆盖设置以候选状态提交；存储失败不会污染共享内存，外部副作用会回滚。
- 壁纸级暂停和停止一次更新全部匹配显示目标，避免多屏逐项命令产生部分成功。
- 托盘与系统监视器的全局/单目标暂停同样使用候选状态，并在播放器或存储失败时回滚。
- 播放和显示器刷新只恢复变更前实际运行且非错误的目标，错误目标不会阻断其他正常目标回滚。
- 关于页显示作者 `NiceBlueChai` 和联系邮箱 `bluechai@qq.com`。
- 关于页仅在用户主动点击后，将 `https://github.com/NiceBlueChai/wall` 交给系统默认浏览器。
- Rust 状态机、三文件原子持久化、暂停原因集合、托盘、单实例和开机启动开关。
- WorkerW 桌面嵌入、mpv 命名管道控制、Cover/Contain/Stretch、九种画幅、三级抗锯齿、
  源/24/30/60 FPS、硬件解码、音量和静音。
- 显示器独立、复制和铺展播放；复制组只保留一个音源并在漂移超过 100 ms 时校准，铺展使用虚拟桌面联合区域。
- 全屏、最大化、电池和显示器休眠自动暂停；手动暂停不会被自动条件误恢复。
- 显示器热插拔保留离线分配，重新连接后恢复；托盘和主界面共享同一目标状态机。
- 同一目标的视频、mpv 路径、硬件解码和显示布局兼容时，切换通过 `loadfile replace` 复用现有 mpv；
  图片/视频互换、硬件解码或布局变化仍安全重启播放器。
- 所有写命令由独立门闩串行化，并只在候选 `WallCore` 上修改；持久化与播放器操作成功后才替换共享状态。
  可失败的播放器副作用会恢复旧快照，不可失败的停止操作则在候选状态持久化后执行。
- 单一后台发布线程按提交顺序发送界面事件并刷新托盘；发布通道失败只写日志，不把已落盘的成功操作误报为失败。
- 自动暂停和目标声音命令在提交状态前显式释放播放器锁，前台播放命令不会再与后台监视器形成锁反转。
- 热切换逐项检查旧媒体、设置、暂停和声音回滚；回滚不完整时停止并丢弃损坏组，由干净重启或监视器恢复。
- 只有开机启动开关实际变化时才访问系统启动项；画面、声音和性能设置不依赖无关的注册表操作。
- 本地资源协议仅动态放行已导入文件；Wall 进程没有商城、账号、遥测、更新检查或 HTTP 请求。

## 数据与迁移契约

- `library.json` 同时保存 `items` 和 `categories`；读取旧版媒体数组时自动迁移为空分类集合。
- `WallpaperItem.settings` 保存可空覆盖项，未设置字段始终继承最新全局设置。
- `settings.json` 保存显示模式和所选显示器 ID；`session.json` 保存每个显示目标、媒体、声音和暂停原因。
- 独立模式目标键由单显示器 ID 决定；复制和铺展目标键由模式及排序后的显示器 ID 决定，保证重启后稳定。
- 原子批量操作先验证所有媒体和分类 ID，任何一项无效时不写入部分结果。

## 开源发布契约

- 源码、项目图标和生成的示例媒体可以按根目录 MIT License 再分发。
- 便携包必须同时包含中英文 README、本地图片、README 引用文档、`LICENSE`、第三方声明和示例媒体说明。
- 发布脚本只能打包项目生成或具有明确再分发许可的媒体，不能读取开发者下载目录。
- FFmpeg 只在构建时生成测试图案，不进入便携包；mpv 运行文件按其独立许可声明分发。
- CI 负责前端测试、前端构建、Rust 格式和 Rust 测试；Windows 原生便携构建仍由发布脚本完成。

## 已知边界

- 当前机器完成了 Windows 实机验证；仍需在独立 Windows 10 和 Windows 11 干净系统上做发布前矩阵验证。
- 媒体尺寸、时长和离线缩略图探测尚未接入；当前列表和详情直接使用 WebView 本地媒体预览。
- 后台每秒检查 mpv 子进程并在意外退出后重建对应显示目标；IPC 读取使用两秒硬超时，避免监视线程永久阻塞。
- 系统默认浏览器的实际启动仍需在发布前做一次人工冒烟验证；自动化只验证按钮、命令地址和原生集成编译。
- 当前开发机只有一台在线显示器；复制、铺展、热插拔和不同 DPI 的发布验收仍需在双屏/三屏 Windows
  10/11 设备上完成，单元测试不能替代真实显卡与桌面组合器验证。

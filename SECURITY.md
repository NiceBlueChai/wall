# 安全策略

## 受支持版本

安全修复优先应用到最新的 v1.x 版本。尚未正式发布或已经停止维护的开发快照不单独提供补丁。

## 报告漏洞

请不要在公开 Issue 中披露尚未修复的漏洞。发送邮件至 `bluechai@qq.com`，主题以 `[Wall Security]` 开头，
并尽量包含：

- Wall 版本、Windows 版本和安装方式。
- 可重复的操作步骤和预期影响。
- 必要的日志或截图；请先删除用户名、路径和其他隐私信息。
- 已知缓解方式，以及是否已经公开披露。

维护者确认问题后会根据影响范围安排修复和披露。项目不承诺固定响应时限，但会避免在补丁发布前公开
可直接利用的细节。

## 安全边界

- Wall 只应读取用户明确导入的本地媒体。
- 应用不得上传媒体、发送遥测或隐式访问网络。
- 本地资源协议只能放行已导入媒体和应用自身资源。
- mpv 以禁用用户配置和交互控件的参数启动，通过本地命名管道控制。
- 日志、崩溃信息和复现文件仍可能包含本地路径，分享前必须人工检查。
